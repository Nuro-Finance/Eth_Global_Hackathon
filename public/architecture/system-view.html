System-wide flow

The 30,000-foot view of AFI. Five logical tiers — client, Next.js edge, Express backend, Postgres, and external services — plus on-chain infrastructure across 23 chains. Every arrow is a real dependency; if you cut it, something breaks.

What each tier does

  • Client — Next.js 15.5 single-page app at app.nuro.finance, wallet connectors (wagmi for EVM, Privy for social login prep).
  • Next.js Edge — serves /public/*.html static files directly (dashboards), and proxies /api/* to the backend without exposing secrets.
  • Express Backend — six major subsystems: REST API (nuro-routes.ts), chain monitor, swap engine, bridge router, Mythos growth agent, and the admin console.
  • Postgres — single source of truth for all intent-layer state. 25 migrations applied. Contains users, cards, transactions, execution_log (every sweep logs), erc20_allowlist (DB-backed since Session 23), post_engagement (time-series metrics).
  • External services — grouped by purpose: money-routing (0x, Circle CCTP, LayerZero, SD3 Issuer), content platforms (Telegram, Moltbook, Twitter, HeyGen), and AI/data (Claude API, CoinGecko, Polymarket CLOB).
  • On-chain — 23 chains total: 22 EVM (Ethereum, Base, Arbitrum, Optimism, Polygon, BSC, Avalanche, Linea, Scroll, Unichain, World Chain, Sonic, HyperEVM, zkSync, Celo, Gnosis, Sei, XDC, Codex, Ink, Plume, Monad) plus Solana via Circle Bridge Kit.
