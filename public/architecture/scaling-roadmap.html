13 · Scaling roadmap

Capacity ladder — 1K → 10K → 100K DAU

Holding treasury constant (covered by the parallel automation track), this is what the software infrastructure can carry today, and what topology shift each tier demands. Three real architecture changes, not one continuous scale curve. Cost goes up by ~10× per tier; effort goes up by ~5×; complexity goes up by ~3× and never comes down.

Migrated to Supabase Pro ($25/mo) — managed Postgres 17.6 on AWS US-East-1 with built-in PgBouncer (transaction pooler at port 6543, IPv4 add-on enabled). 169,378 rows transferred clean across 10 critical tables. SSL auto-detected via connection string regex; rejectUnauthorized:false for self-signed cert chain.

Tier 1 mitigations applied:

  • ✓ Managed Postgres via Supabase (replaces self-hosted Postgres on VPS)
  • ✓ Built-in PgBouncer transaction pooler (Tier 2 connection-pooling capability obtained early at no extra cost)

  • ✓ pm2-logrotate setup script committed (scripts/setup-logrotate.sh) — 10MB max, daily, 30-day gzip retention. Run-once on VPS.

+ Added (vs Tier 1)
  • PgBouncer in transaction-pooling mode — 200–500 effective conns from a single PostgreSQL backend
  • Postgres read replica — physical replication, separate VM (~$50–100/mo). Routes /agents/:id/details + /notifications + analytics

− Removed (vs Tier 1)

  • Notification 30s polling from FE replaced by SSE

+ Added (vs Tier 3 managed)
  • Bare-metal servers — 4 boxes (compute, DB primary, replica+cache, backup). ~$15–40K upfront
  • Colocation contract — 1/4 rack at a Tier 3+ datacenter. ~$300–800/mo. Includes power, cooling, cross-connect
  • Self-managed Postgres — full control of tuning, extensions, version upgrades; physical streaming replication
  • Redis Sentinel cluster — 3-node HA, keepalived VIP for automatic failover
  • ZFS storage pool — nightly snapshots, off-site rsync to Backblaze B2 ($5/TB/mo)
  • Self-hosted observability — Grafana + Loki + Tempo on Box A. No Datadog/Honeycomb subscription
  • Owner-operator skill — someone who can debug a kernel issue at 3am
− Removed (vs Tier 3 managed)
  • Managed Postgres bill — Supabase/RDS bill goes to zero (~$500–2K/mo savings)
  • Managed Redis bill — Upstash Pro bill goes to zero (~$50–500/mo savings)
  • k8s / Fly.io / Render — replaced by Docker Compose or Nomad on bare metal
  • Datadog / Honeycomb subscription — replaced by self-hosted Grafana stack (~$100–500/mo savings)
  • Cloud egress fees — colocation deals usually include generous transit; cross-AZ replication doesn't cost cents/GB
  • Provider lock-in — your data, your hardware, your control plane
~ Trade-offs
  • YOU are now responsible for hardware failure, RAID rebuilds, kernel patches, datacenter trips
  • Capacity is fixed at the rack — scaling beyond requires more boxes (lead time + colo expansion)
  • No multi-region — single colocation = single point of failure unless you do dual-colo
  • CDN still external (Cloudflare or similar) — colo isn't optimal for static asset edge delivery
  • External backup target (Backblaze B2 / off-site) remains essential
Capex: $15–40K hardware upfront Opex: ~$500–1K/mo (colo + electricity + bandwidth + B2 backup) Hardware: 4× rack-mount servers (compute + DB primary + replica/cache + backup) ROI vs cloud: ~14–20 months at sustained 100K DAU

3Treasury automation — parallel track

Treasury automation unlocks every tier

Software capacity is necessary but not sufficient — the chains run dry long before the DB does. Per-chain reserve targets + a 5-min refill cron with gas-price gating keep the system funded automatically. Without this, even a fully-scaled Tier 3 stack halts on a quiet weekend. Full plan in Marathon 10. Buildable today in OBSERVE-only mode — flips live the moment funding arrives.
