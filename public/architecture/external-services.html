05 — Defense in depth

Every external call is wrapped

Each outbound request is in a try/catch with structured error logging to execution_log. Rate-limit responses and transient 5xx bubble up as task-level failures; they don't cascade into retry storms. The gates.yaml registry (9 gates) encodes the preconditions for flipping risky flags — run /gate-check <name> before any .env edit or VPS deploy.

Color-coding recap

  • 🟢 Live (green border) — currently active in production
  • 🟡 Dormant (amber border) — keys missing, feature dormant until set
  • 🔴 Gated off (red border) — intentionally disabled behind a flag; re-enable via /gate-check workflow
