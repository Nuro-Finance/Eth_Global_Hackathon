10 · Deployment

Deployment topology

Dev machine → GitHub → VPS. Both backend (~/Nuro-Finance) and frontend (~/Nuro-Finance) live on the same VPS at 74.50.109.203 under PM2. Three pre-push gates protect main: verify-deps (all imports declared in package.json), typecheck (backend scope only), 84 vitest tests. Migration-first rule: schema changes always apply before the backend restart.

04 — Runtime state

Monitor polling state

Paused by default at POLL_INTERVAL_MS=86400000 (24h). Stop hook in ~/.claude/settings.json auto-pauses when Claude Code sessions end (prevents accidental Alchemy spend). Un-pause via:

sed -i 's|POLL_INTERVAL_MS=.*|POLL_INTERVAL_MS=60000|' ~/Nuro-Finance/.env && pm2 restart 4 --update-env

PM2 process IDs (don't guess, look them up)

  • id 4 — nuro-api — backend, port 3000, tsx runtime
  • id 1 — nuro-frontend — Next.js SSR, port 2800

Confirm with ssh nuro@74.50.109.203 "pm2 list | head -8" before restarting anything.

Nginx routing (Session 25 additions)

  • app.nuro.finance/admin → backend port 3000 (admin console + auth gate) — live as of Session 25
  • app.nuro.finance/telegram/webhook → backend port 3000 (replaces 10s polling) — nginx ready; setWebhook call still pending
  • app.nuro.finance/ → frontend port 2800 (Next.js SSR) — default
  • api.nuro.finance/ → backend port 3000 (REST API) — legacy direct path

Config source lives in the repo at ops/nginx-nuro.conf; installer at ops/install-nginx-nuro.sh.

Why the migration-first rule matters. A schema change that hasn't been applied yet means the new backend code will hit "column does not exist" on first query. Always apply migrations before restart — otherwise the restart will cause downtime while you figure it out.
