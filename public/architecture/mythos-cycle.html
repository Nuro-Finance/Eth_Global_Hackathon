04 · Growth agent

Mythos daily cycle

The state machine that powers our autonomous content agent. Perceives the market, thinks through options via Claude, generates content, runs it through quality + risk gates, queues medium/high-risk posts for Telegram approval, publishes, and (starting Session 24) seeds an engagement snapshot so future cycles can learn from what performed.

stateDiagram-v2 [*] --> Perceive: daily cron fires Perceive: Perceive market Perceive --> Think Think: Think via Claude Think --> Generate Generate: Generate content Generate --> Gate Gate: Quality gate Gate --> Risk Risk: Classify risk Risk --> AutoApprove: low Risk --> Queue: medium or high AutoApprove: Auto-approved AutoApprove --> Publish Queue: Queue for Telegram Queue --> Wait Wait: Wait for admin tap Wait --> Publish: Approve Wait --> Reject: Reject Wait --> Expire: 2h timeout Publish: Post to platform Publish --> Seed Seed: Seed engagement row Seed --> Track Track: Hourly metrics fetch Track --> Learn Learn: Weight next cycle Learn --> [*] Reject --> LogRej LogRej: Log rejection LogRej --> [*] Expire --> [*]

Where each step lives

  • Perceive — src/growth-agent/skills/thought-engine.ts::perceiveMarket pulls CoinGecko prices, Polymarket trending, recent sports events.
  • Think + Generate — thought-engine.ts runs Claude with a system prompt keyed to Mythos's identity, produces structured content candidates (crypto alert / market prediction / educational thread).
  • Gate — quality-gate.ts rejects boilerplate, dupes, or content that fails the 5-criteria quality check.
  • Risk — classifier scores posts low/medium/high based on claims, topic sensitivity, financial advice proximity.
  • Queue + Wait — approval-pipeline.ts posts Telegram message with inline Approve/Reject buttons. Session 23 moved the approval poll from 60-min to 10-second intervals.
  • Publish — processApprovedPosts routes to moltbook.ts, twitter.ts, or telegram.ts per platform.
  • Seed + Track — on publish, seedPostEngagement writes a t=0 row. Session 24's engagement-fetcher.ts cron samples metrics hourly.
  • Learn — Session 25+ scope. Query post_engagement by (format, tone, category) to weight future generation toward high-engagement patterns.

Why the Telegram approval matters

Mythos isn't fully autonomous. Medium/high-risk posts (financial claims, contentious takes) require admin approval before they go live. The 10-second polling interval means tap-to-publish latency is ~10-20s, not 60+ min like before the Session 23 fix.
