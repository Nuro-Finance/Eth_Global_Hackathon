Nuro on-chain · proof of existence

Every Nuro-deployed contract across the chains we support, with block-explorer links for verification. Contracts cover the LayerZero V2 OFT bridge (synthetic USDC routing where CCTP isn't available), fee vaults (collected protocol revenue), and x402 agentic-finance vaults (revenue from external agent traffic). Everything is HD-derived from a single root key for clean attribution.

Why 5 chains share the same address

CREATE2 deterministic deployment: same deployer address + same salt + same bytecode produces the same address on every EVM chain. Base, Celo, Gnosis, Scroll, and zkSync Era all received their adapters via this path. Proof we're using a single canonical deployment pipeline. Arbitrum and BSC differ because they were deployed earlier with non-CREATE2 paths.

Chains supported via external public infrastructure

These chains do not require Nuro-deployed contracts because they use mature public infrastructure (Circle CCTP V2, Across Protocol, Bridge Kit) for routing. Each user gets an HD-derived deposit address, but no bespoke smart contract on-chain.
