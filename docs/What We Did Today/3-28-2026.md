## Phase 1: UUID on Card Graphic — Root Cause + Fix

### The Problem
The Owen card UUID (`77d9fb1e-bdf7-461b-a8fe-e73954120dd9`) was rendering as text in the top-right corner of every card on the card graphic, where the card network label (e.g. "VISA") should appear.

### Root Cause Trace
The `CreditCard` component accepts an `id` prop used for the network label display. The prop chain was:

```
CardSection/index.tsx  →  CardStack  →  CreditCard  →  CardContent
apiCards.map(c => ({ id: c.id, ... }))              renders {id} top-right
```

`c.id` is the DB UUID (like `77d9fb1e-...`). It was being spread through as the network label instead of "VISA".

### Files Changed

**`CardSection/index.tsx`** — map `cardType` to display label:
```tsx
// Before:
id: c.id,
// After:
id: c.cardType === "VIRA" ? "VISA" : (c.cardType || "VISA"),
```

**`CardStack/index.tsx`** — decouple React key from the display label:
```tsx
// Before:
key={card.id}
// After:
key={`card-${originalIndex}`}
```

**`cardStack.config.ts`** — fix demo card network labels:
```
id: "vira"      → id: "VISA"
id: "masteecard" → id: "MC"
```
