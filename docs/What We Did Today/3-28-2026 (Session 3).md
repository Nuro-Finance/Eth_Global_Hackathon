## Phase 2: Static Prerender Failures

**Problem**: After downgrade, build crashed at static page generation. `"use client"` pages with auth hooks crash during prerender (no provider context during static generation).

**Error**: `Cannot destructure property 'status' of undefined`

**Fix**: Added `export const dynamic = "force-dynamic"` to `src/app/[locale]/layout.tsx` (Server Component). Cascades to all child routes, prevents static prerender.

## Phase 3: Runtime SSR Auth Crash (Context Window 2)

**Problem**: Build passed but `/en/dashboard` showed "Application error: client-side exception". The `force-dynamic` prevented build-time prerender but Next.js still SSRs pages at runtime. `ProtectedRoute` called `useSession()` from `next-auth/react` which returned `undefined` during SSR ‚Äî no `SessionProvider` existed anywhere in the codebase.

**Error**: `TypeError: Cannot destructure property 'status' of '(0 , e.wV)(...)' as it is undefined.`

**Failed fixes**:
- Safe destructuring (`session?.status ?? "loading"`) ‚Üí permanent loading state (useSession always undefined without provider)
- Switched to `usePrivy()` ‚Üí wrong auth system entirely (NEXT_PUBLIC_PRIVY_APP_ID not set, Privy provider disabled)

**Root cause discovered**: App uses NextAuth v5 (`next-auth@5.0.0-beta.30`) with `src/auth.ts` exporting `{ handlers, auth, signIn, signOut }`. The `ProtectedRoute` correctly used `useSession()` from `next-auth/react`, but NextAuth v5's `useSession()` requires `SessionProvider` wrapper. Chris's `Providers.tsx` never included one.

**Fix** (two parts):
1. Added `SessionProvider` from `next-auth/react` to `src/providers/Providers.tsx` wrapping `ReduxProvider`
2. Added `mounted` state guard to `ProtectedRoute` to skip auth check during SSR

## Key Lessons

1. **Next.js 16 + Privy = Turbopack incompatibility** ‚Äî `thread-stream` via pino crashes Turbopack. No fix. Stay on Next.js 15.
2. **`export const dynamic` only works from Server Components** ‚Äî `"use client"` files silently ignore it.
3. **NextAuth v5 `useSession()` requires `SessionProvider`** ‚Äî unlike v4 which could work without it in some configs.
4. **Chris's visual upgrade shipped with mock data everywhere** ‚Äî the `transactionSlice.ts` fetchTransactions thunk is fake. The `useTransactionsState` hook in TransactionsGrid IS properly wired to the API but falls back to mock when not authenticated.
5. **Chris's sidebar config references routes that don't exist** ‚Äî `my-card-1` has a feature directory but no route page. Backend was never touched during this session.
