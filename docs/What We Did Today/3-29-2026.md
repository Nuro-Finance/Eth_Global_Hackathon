## Login System — Fixed
- **Root cause**: `authSlice.loginUser` was checking hardcoded demo credentials (`admin@dashboard.com / Admin@123`) and NEVER calling the real backend
- **Fix**: Rewrote `useLogin.ts` to call NextAuth's `signIn("credentials", ...)` instead of Redux `loginUser`
- NextAuth's `authorize()` in `auth.ts` already calls real backend `POST /auth/login` — just wasn't being used
- Added `/api/auth/login` proxy route
- Fixed `router.push("/dashboard")` → `router.push("/en/dashboard")` (locale prefix required)
- Login now works with real credentials

## Card Data — Connected to Real DB

- `handleCardNameChange` persists `PATCH /api/cards/${cardId}` with `card_name`
- Added `POST` handler to `/api/cards/route.ts` proxy

## Card Controls Persistence — Fixed
- **Bug 1**: `useCardControls.ts` had escaped template literals (`Bearer \${token}`) — auth headers were literal strings, all API calls silently failed
- **Bug 2**: `handleCardNameChange` in `useCardsState` wrote to `cardType` in React state instead of `cardName`
- **Bug 3**: `CardsGrid/index.tsx` initialized cardName from `selectedCard.cardType` ("VISA") instead of `selectedCard.cardName`
- **Bug 4**: My Card page (`my-card-v2`) had `cardName` hardcoded to `"My Card"`, never read from DB
- **Fix**: Added `handleSaveCardName` function that PATCHes backend, syncs from `selectedCard.cardName`, falls back to `cardType`
- Card name now persists across both My Card and Agent Cards pages
