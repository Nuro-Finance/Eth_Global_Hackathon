## Phase 4: Wallet-1 White Page Fix

**Root cause**: `wallet-1/index.tsx` called `usePrivy()`, `useConnectWallet()`, `useCreateWallet()` at top level. When `NEXT_PUBLIC_PRIVY_APP_ID` is not set, no real `PrivyProvider` is in the tree, so these hooks throw "usePrivy must be used within PrivyProvider", crashing the page white.

**Fix**: Split into two components:
- `Wallet1Feature` (shell) — calls only `usePrivyRuntime()` to check `privyEnabled`
- `WalletPrivyContent` (inner) — calls all Privy hooks, only rendered when `privyEnabled=true`
- When `privyEnabled=false`: shows "Wallet Coming Soon" graceful fallback state
