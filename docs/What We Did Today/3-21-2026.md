**Owens contracts endpoint:** `GET /users/{owen_user_id}/contracts` returns contracts per chain, each with `tokens[].balance`. Base chain (chainId 8453) USDC token `0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913` shows the real spendable balance.

**Added to `nuro-routes.ts`:**
```
GET /cards/balance — fetches live from Owens contracts API, sums all token balances, syncs to local DB, returns { balance: 21.928465 }
```

**Frontend:** `useAccountBalance.ts` already used `useCardsState()` to sum card balances — no frontend change needed. The `/cards` endpoint now returns the Owens-synced balance.

## Phase 8: Real Bridge Transactions

**Problem:** 7 seeded fake transactions existed in the DB ($150k etc). These were deleted. The real data lives in the `transactions` table, which the bridge monitor populates on every confirmed CCTP transfer.

**transactions table schema:**
```
id, user_id (= owen_user_id), user_wallet, base_deposit_address,
source_chain, dest_chain, token, amount, fee, forwarded, route,
tx_hash, status, timestamp, created_at
```

**Added to `nuro-routes.ts`:**
```
GET /transactions — looks up owen_user_id for authenticated user, queries transactions table, returns mapped array
```

Richard Wayne has 35 real confirmed USDC bridge transactions in the DB, totaling ~$494 USDC lifetime deposits.

**Frontend:**
- `useTransactionsState.ts` updated to `Promise.all([/card-transactions, /transactions])` — merges and sorts both sources by date
- `TransactionsPanel/index.tsx` — removed `{ t }` argument from hook call
- Import path fixes across the hook file
