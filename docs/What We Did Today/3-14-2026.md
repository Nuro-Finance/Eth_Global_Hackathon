## What We Built Today

**Phase 1: LayerZero Hybrid Architecture**

Deprecated the old synthetic OFT architecture (OFTAdapter only on Base, synthetic MyOFT everywhere else). Deployed fresh `MyOFTAdapter` contracts on both ends of every pathway so only real USDC flows, no synthetics.

|Chain|Contract|Address|
|---|---|---|
|Arbitrum (LZ relay hub)|MyOFTAdapter|`0xd58C1412e50fF00212770B170D86e2387D2d2b18`|
|zkSync Era|MyOFTAdapter|`0xA150EC8B718C22E12036f916d90FF72af14B3E96`|
|Scroll|MyOFTAdapter|`0xA150EC8B718C22E12036f916d90FF72af14B3E96`|

Wired all four peer relationships directly via `setPeer()` calls, bypassing the broken `generateConnectionsConfig` helper which lacked DVN metadata for zkSync and Scroll.

**Phase 2: Bridge Architecture**

Rewrote `bridge.ts` to support three routes:

1. **CCTP V2 direct** for all Circle-supported chains (Ethereum, Arbitrum, Optimism, Polygon, Avalanche, HyperEVM, Linea, Unichain, Base)
2. **LZ two-hop** for non-CCTP chains (zkSync, Scroll): LZ bridges to Arbitrum relay hub, then CCTP carries it to Base
3. **Base direct** for deposits already on Base

Dumped the Circle SDK entirely after discovering `@circle-fin/provider-cctp-v2@1.4.0` had broken ethers v5/v6 incompatibility. Replaced with raw on-chain CCTP V2 calls using `depositForBurn` on `TokenMessengerV2` + Iris API attestation polling.

**Phase 3: Deposit Monitor**

Built `monitor.ts`, a polling loop that runs every 15 seconds watching every user's deposit address across all 8 supported chains. When it detects a new USDC balance it automatically triggers the full bridge pipeline without any manual webhook needed.

**Phase 4: Owen Webhook Integration**

Updated the `/webhook` endpoint to parse Owen's SD3 payload format:

json

```json
{ "resource": "transaction", "body": { "collateral": { "userId", "amount", "chainId" } } }
```

Amount unit confirmed as cents (9500 = $95.00).

**Phase 5: End-to-End Live Test**

Deposited $1 USDC on Arbitrum to Cashly deposit address `0x75Aa3B70Cb3885c860246C8d5c5103368a9c45fC`. Monitor detected it automatically. CCTP burned on Arbitrum, Circle attested after ~15 minutes, auto-relayed to Owen's Base contract `0x34e81c59B814874611C7FB66661B57E599b4857D`. Card balance went from $17.38 to $18. Confirmed on the Cashly app.

## Errors Fixed Along the Way

|Error|Fix|
|---|---|
|`generateConnectionsConfig` DVN lookup fails for zkSync/Scroll|Bypassed entirely, used direct `setPeer()` calls|
|Circle SDK `INPUT_UNSUPPORTED_ROUTE`|SDK's `supportsRoute()` called without token arg internally|
|Circle SDK `ethers.JsonRpcProvider is not a constructor`|ethers v5/v6 mismatch, SDK requires v6|
|Circle SDK `invalid signer or provider`|Same v5/v6 conflict, unfixable without major refactor|
|Replaced Circle SDK entirely with raw CCTP V2|`depositForBurn` + Iris polling, works with ethers v5 natively|
|Nonce collision on retry|Added explicit `provider.getTransactionCount("latest")` before each tx|
|Allowance reuse|Added allowance check, skip approve if already sufficient|
|Monitor double-trigger|`lastSeen` cache prevents double processing within 5 min window|
